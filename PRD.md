# 产品需求文档 (PRD)

## 1. 产品概述
**产品名称**: AI 驱动的文字冒险游戏 (Whatif)
**产品愿景**: 打造一个完全由 AI 实时生成的、具有高度沉浸感和无限可能性的文字冒险游戏。通过结合大语言模型 (LLM)、图像生成模型和视频生成模型，为玩家提供独一无二的视听和叙事体验。

## 2. 核心功能需求
### 2.1 游戏设置与初始化
- **语言选择**: 支持中文和英文界面及剧情生成。
- **API 配置**: 支持配置 LLM (Gemini/DeepSeek)、图像生成 (Gemini/Doubao) 和视频生成 (Doubao) 的 API Key。
- **世界观设定**: 玩家可以自定义输入世界观背景，或使用系统提供的随机预设。
- **角色设定**: 玩家可以设定角色名称、性别和性格特征（特质）。

### 2.2 核心玩法系统
- **动态叙事**: AI 根据当前状态和玩家选择，实时生成剧情、场景描述和 3 个分支选项。
- **状态管理 (HP)**: 初始 80，受行动类型、违规惩罚、时间流逝、天气和道具影响。归零则触发坏结局。
- **规则系统**: 每个章节新增一条世界规则，玩家若在选项中违规将受到严厉的 HP 惩罚。
- **物品与背包**: 玩家在探索中可获得道具，并在特定场景使用以改变状态或抵御负面效果。
- **动态地图**: 采用图 (Graph) 结构，包含当前节点、已访问节点和未知节点，支持节点间的移动。
- **天气与时间**: 随回合推进动态变化，影响探索难度和状态消耗（如夜晚和极端天气会加速 HP 消耗）。

### 2.3 视觉与多媒体生成
- **双重分镜系统 (Visual Sequence)**: 每次剧情推进时，AI 自动生成包含视觉锚点（角色特征、画风）和动作反馈（动作过程、结果场景）的 `visual_sequence`。
- **场景图像生成**: 提取 `visual_sequence` 生成场景图，如果 API 支持，将上一帧作为参考图传入以保持连贯性。
- **动态视频生成**: 在生成图像后，后台静默调用视频模型，将静态图像（首帧）、动作描述（过程）和结果图像（末帧）转化为动态视频，增强沉浸感。

### 2.4 章节与结局
- **章节推进**: 游戏分为多个章节，每个章节包含若干个行动回合 (Step)。
- **多结局系统**: 根据玩家的违规次数、最终状态值，触发好结局 (Happy)、坏结局 (Bad) 或开放结局 (Open)。
- **通关回顾 (Storyboard)**: 游戏结束后，玩家可以查看整个游戏过程的分镜故事板（包含文本、图像和生成的视频）。

## 3. 用户交互流程
1. **启动**: 进入语言和 API 设置。
2. **创角**: 设定世界观和角色。
3. **游玩**: 
   - 阅读 AI 生成的剧情和场景图。
   - 查看当前状态、地图和背包。
   - 从 3 个选项中选择行动，或使用背包道具。
4. **结算**: 达成结局条件，查看故事板回顾。
