# 产品需求文档 (PRD)

## 1. 产品概述
**产品名称**: AI 驱动的文字冒险游戏 (Whatif)
**产品愿景**: 打造一个完全由 AI 实时生成的、具有高度沉浸感和无限可能性的文字冒险游戏。通过结合大语言模型 (LLM)、图像生成模型和视频生成模型，为玩家提供独一无二的视听和叙事体验。
### 1.1 背景与痛点
在传统的视频制作、动画预演和概念设计中，将“假设性文本（What-If scenarios）”转化为具备物理逻辑和空间连贯性的视觉分镜，往往需要耗费大量的人工介入、3D建模和反复调整。现有的文本到视频（Text-to-Video）模型多侧重于画面表现力，而在**对象状态的物理逻辑连贯性**与**复杂空间推理**上表现不足。
### 1.2 核心价值主张 (Value Proposition)
`What-If 视频生成框架` 是一个由自然语言驱动的**仿真模拟 (Emulative Simulation)** 系统。本框架致力于自动化处理空间推理与叙事分镜的生成，通过严格的“对象状态改变 (OSC)”评估引擎，确保生成的视觉内容在物理和逻辑层面上高度符合用户的假设条件，极大降低从“脑暴假设”到“视觉分镜”的转化成本。

## 3. 核心功能需求
### 3.1 游戏设置与初始化
- **语言选择**: 支持中文和英文界面及剧情生成。
- **API 配置**: 支持配置 LLM (Gemini/DeepSeek)、图像生成 (Gemini/Doubao) 和视频生成 (Doubao) 的 API Key。
- **世界观设定**: 玩家可以自定义输入世界观背景，或使用系统提供的随机预设。
- **角色设定**: 玩家可以设定角色名称、性别和性格特征（特质）。

### 3.2 核心玩法系统
- **动态叙事**: AI 根据当前状态和玩家选择，实时生成剧情、场景描述和 3 个分支选项。
- **状态管理 (HP)**: 初始 80，受行动类型、违规惩罚、时间流逝、天气和道具影响。归零则触发坏结局。
- **规则系统**: 每个章节新增一条世界规则，玩家若在选项中违规将受到严厉的 HP 惩罚。
- **物品与背包**: 玩家在探索中可获得道具，并在特定场景使用以改变状态或抵御负面效果。
- **动态地图**: 采用图 (Graph) 结构，包含当前节点、已访问节点和未知节点，支持节点间的移动。
- **天气与时间**: 随回合推进动态变化，影响探索难度和状态消耗（如夜晚和极端天气会加速 HP 消耗）。

### 3.3 视觉与多媒体生成
- **双重分镜系统 (Visual Sequence)**: 每次剧情推进时，AI 自动生成包含视觉锚点（角色特征、画风）和动作反馈（动作过程、结果场景）的 `visual_sequence`。
- **场景图像生成**: 提取 `visual_sequence` 生成场景图，如果 API 支持，将上一帧作为参考图传入以保持连贯性。
- **动态视频生成**: 在生成图像后，后台静默调用视频模型，将静态图像（首帧）、动作描述（过程）和结果图像（末帧）转化为动态视频，增强沉浸感。视频播放完毕后自动提取最后一帧作为下一回合的背景图。
- **通关视频合并**: 游戏结束后，在“故事板”界面自动将所有生成的视频片段拼接成一个完整的通关视频，供玩家回顾和保存。

### 3.4 章节与结局
- **章节推进**: 游戏分为多个章节，每个章节包含若干个行动回合 (Step)。
- **多结局系统**: 根据玩家的违规次数、最终状态值，触发好结局 (Happy)、坏结局 (Bad) 或开放结局 (Open)。
- **通关回顾 (Storyboard)**: 游戏结束后，玩家可以查看整个游戏过程的分镜故事板（包含文本、图像和生成的视频），并自动合并生成完整的通关大片。

## 4. 用户交互流程
1. **启动**: 进入语言和 API 设置。
2. **创角**: 设定世界观和角色。
3. **游玩**: 
   - 阅读 AI 生成的剧情和场景图。
   - 查看当前状态、地图和背包。
   - 从 3 个选项中选择行动，或使用背包道具。
4. **结算**: 达成结局条件，查看故事板回顾。

## 5. 验收标准 (Acceptance Criteria)
为确保项目质量达到预期，以下为核心流程的验收基准：
1.  **OSC 逻辑通过率**：在标准的 50 个复杂物理假设测试集中，框架对“对象状态演变”的推演准确率需达到 90% 以上（由人工或评估模型打分）。
2.  **工作流完整性**：用户能够不中断地完成“文本输入 -> 逻辑确认 -> 分镜生成 -> 局部微调”的端到端闭环。
3.  **UI 表现层同步**：前端 React 界面需准确无误地实时反映后台全局状态库 (Global Store) 的每一次 FSM 状态扭转。



## 6. 未来迭代规划 (Roadmap)
* **Phase 1 (MVP)**：完成核心层级架构搭建，跑通纯文本的“What-If”空间逻辑推演闭环。
* **Phase 2**：接入主流文生图/文生视频 API，实现基于推演逻辑的自动化分镜渲染，完善 OSC 拦截机制。
* **Phase 3**：引入 3D 空间坐标信息支持，允许导出数据至外部 3D 软件（如 Blender、Unreal Engine）进行二次开发。
